## Briefly; Context Engineering (v1)

### 제품 한 줄 정의
**Briefly; = 전 세계 시장은 우리가 읽고, 고객은 하루 한 통의 개인 맞춤 경제레터(웹/메일)만 팔로업한다.**

### 대상 시장/범위 (MVP)
- **Universe**: **S&P 500 (미국만)**
- **핵심 진입**: Daily(오늘) + Market Heatmap

### 역할(Role)
- **Admin/Editor(운영자)**: 콘텐츠 작성/발행, 개인화 블록 검수/승인, 뉴스 수집/정리
- **Customer(고객)**: 콘텐츠 열람, 온보딩 정보 입력, 포트폴리오 입력(옵션), BUY/SELL 로그 작성

### 핵심 상품(매일)
- **Daily = 운영자 공용 글 1개(하루 1개 고정) + 고객별 개인화 블록 3개**
- **급변장 대응**: Daily 추가 발행 없음 → **Market Blog에 업데이트 글 발행**
- **Daily 아카이브**: 웹에서 **날짜별 재열람 가능(저장 O)**

### 제공 기능(고객 관점 IA)
- **Today (Daily)**: 공용 Daily + 개인화 3블록 + 날짜별 아카이브
- **Heatmap (S&P 500)**: 1일 등락률 기반 색상 → 종목 클릭 시 Ticker Hub 이동
- **Ticker Hub (기업 페이지)**: **뉴스 먼저**, 이후 운영자 리서치 글 아카이브(최신순)
- **Portfolio**: 포트폴리오 서클(비중) + 리스트(상세)
- **Trades**: BUY LOG / SELL LOG (왜 샀/팔았는지)
- **Market Blog**: 운영자 관점/급변장 업데이트
- **Weekly Blog**: 주말 1회
- **Setups**: Stock/Sector to watch
- **Library/Handbook**: 준비중(목차형)

### 클릭/탐색 규칙(중요)
- Heatmap 종목 클릭 → **Ticker Hub(뉴스 → 리서치)**
- Portfolio 리스트에서
  - **기업 로고 클릭** → 운영자 리서치 글(기업 아카이브)
  - **행의 다른 영역 클릭** → 해당 기업의 내 BUY/SELL 로그 요약

### 요금제(권한)
- **$10 (Basic Letter)**: 커스텀 경제레터 블로그에만 접근
  - Daily / Weekly / Market Blog
- **$30 (Intelligence)**: **모든 것 접근**
  - Basic 포함 + Heatmap + Ticker Hub(리서치 포함) + Setups + Trades + Portfolio + (향후) Handbook
