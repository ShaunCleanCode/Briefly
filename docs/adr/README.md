## ADR (Architecture Decision Records)

### Purpose
Use ADRs to record decisions that are costly to change later:
- stack choices (Next.js/Supabase/etc.)
- API contract conventions
- data retention & consent policy
- E2E testing strategy (real API vs mocks)

### Format
Create files like:
- `0001-decision-title.md`

Each ADR should include:
- Context
- Decision
- Consequences
- Alternatives considered
